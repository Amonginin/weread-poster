# 微信读书热力图生成工具测试计划 (Test Plan)

本测试文档旨在对照 [README.md](file:///d:/coding/personalProj/weread-poster/README.md) 中声称的功能、CLI 参数和环境变量，制定完备的功能验证用例。由于在测试环境中可能缺乏真实有效的微信读书 API 凭证，我们将使用 Python 的 `unittest.mock` 对网络数据进行 Mock，以模拟真实 API 交互，确保测试的完整可重复性。

---

## 1. 测试用例定义

### 1.1 基础与身份鉴权 (TC-01)
* **测试目的**：验证 API Key 缺失时的退出逻辑与鉴权中转的完整性。
* **TC-01-01 (未配置凭证退出)**:
  - **输入**：清空环境变量 `WEREAD_API_KEY`，运行 `python -m weread_poster`。
  - **预期输出**：程序输出“请设置环境变量: export WEREAD_API_KEY=<你的API Key>”并以退出码 `1` 退出。
* **TC-01-02 (凭证校验失败退出)**:
  - **输入**：配置失效的环境变量（如 `wrk-invalid`），Mock `test_auth` 返回校验失败。
  - **预期输出**：程序输出“认证失败...”并以退出码 `1` 退出。

### 1.2 年份解析与过滤 (TC-02)
* **测试目的**：验证命令行中 `--year` 参数在不同形式下的解析可靠性及无数据年份过滤。
* **TC-02-01 (单年份解析)**:
  - **输入**：`--year 2025`。
  - **预期输出**：程序将其解析为区间 `(2025, 2025)`。
* **TC-02-02 (范围年份解析)**:
  - **输入**：`--year 2023-2025`。
  - **预期输出**：程序将其解析为区间 `(2023, 2025)`，绘图器能够为 2023、2024 和 2025 三年绘制对应的年历网格。
* **TC-02-03 (非法年份格式拦截)**:
  - **输入**：`--year 202a`。
  - **预期输出**：抛出 `ValueError` 并被 CLI 优雅捕获退出。
* **TC-02-04 (空数据年份规整)**:
  - **输入**：年份跨度 `2023-2025`，Mock 2023年有数据、2024和2025年完全没有有效阅读时长。
  - **预期输出**：`reduce_year_list` 主动滤去 2024 和 2025 年，生成的 SVG 仅保留 2023 年以保持视图紧凑。

### 1.3 配色主题与覆写定制 (TC-03)
* **测试目的**：验证预设配色主题及各项背景、文本、轨道自定义颜色参数是否在 SVG 树中正确被覆写。
* **TC-03-01 (主题色渲染)**:
  - **输入**：`--theme rose`。
  - **预期输出**：输出的 SVG 中使用的主要渐变电平色与 `rose` 主题的 HEX 相同。
* **TC-03-02 (命令行颜色覆写)**:
  - **输入**：`--background-color "#000000" --text-color "#111111"`。
  - **预期输出**：生成的 SVG 底色矩形填充值被改写为 `#000000`，文字样式颜色变更为 `#111111`。

### 1.4 动态动画与时间切片 (TC-04)
* **测试目的**：验证 SVG 内置不透明度淡入动画节点的生成与动画时间长度配置。
* **TC-04-01 (无动画静态 SVG)**:
  - **输入**：默认参数（不携带 `--with-animation`）。
  - **预期输出**：生成的 SVG 中格点 `<rect>` 元素无 `<animate>` 子节点。
* **TC-04-02 (带动画 SVG 与周期设定)**:
  - **输入**：`--with-animation --animation-time 15`。
  - **预期输出**：每个有数据的格子 `<rect>` 下附加了 `<animate>` 子元素，动画周期 `dur` 属性设定为 `15s`，`keyTimes` 时间片切分精准。

### 1.5 数据保存与统计 (TC-05)
* **测试目的**：验证导出 JSON 格式数据的正确性以及命令行控制台摘要打印。
* **TC-05-01 (JSON 文件导出)**:
  - **输入**：`--json my_raw_data.json`。
  - **预期输出**：本地成功写入 `my_raw_data.json`，且数据字典的值已逆向折算回秒数。
* **TC-05-02 (统计状态打印)**:
  - **输入**：`--stats`。
  - **预期输出**：命令行标准输出流打印“===== 阅读统计摘要 =====”且正确显示总时长、阅读天数与日均信息。

### 1.6 环境变量与优先级 (TC-06)
* **测试目的**：验证环境变量对配置的控制，以及命令行参数优先级是否高于环境变量。
* **TC-06-01 (环境变量配置)**:
  - **输入**：配置环境变量 `THEME_COLOR=rose`，命令行不传 `--theme`。
  - **预期输出**：生成的主题色默认为 `rose`（玫瑰粉）。
* **TC-06-02 (命令行参数覆写环境变量)**:
  - **输入**：配置环境变量 `THEME_COLOR=rose`，命令行显式传递 `--theme purple`。
  - **预期输出**：海报以命令行参数为高优先级，生成 `purple`（梦幻紫）热力图。

## 2. 测试执行策略

我们将创建一个自动化单元测试套件 `tests/test_weread_poster.py`，其中包含使用 Python 的 `unittest` 库针对上述 TC 用例编写的测试代码。
* 网络层使用 Mock 拦截 `WeReadAuth.call_gateway`，使其在收到数据请求时，固定返回 2025 年 6 月共计 3 天阅读数据的 Mock Payload。
* 执行测试脚本：`python -m unittest tests/test_weread_poster.py`。
